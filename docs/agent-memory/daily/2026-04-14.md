# 2026-04-14

## Context
- Main focus: Shopify shell setup UX gap for `Orders flowing` checklist step.
- Environment: local monorepo + production server `167.172.60.234`.

## Decisions
- Keep existing checklist architecture; add explicit action guidance instead of redesigning setup flow.
- Route `Continue setup` for `Orders flowing` directly to an anchored guidance section.

## Changes
- Updated `packages/shopify-app/app/routes/app.tsx`:
  - `Orders flowing` checklist detail now explains what merchant must do.
  - next-step route now points to `/app/integrations#orders-flow`.
  - button label for that step is now `Open order flow setup`.
- Updated `packages/shopify-app/app/routes/app.integrations.tsx`:
  - added new section `Orders flow setup` (`id="orders-flow"`).
  - shows active/pending status based on `overview.metrics.totalOrders`.
  - pending state now includes concrete step-by-step instructions and action buttons:
    - `Go to dashboard`
    - `Refresh order flow status`
  - active state shows confirmation card with order count.
- Build verification:
  - `pnpm --filter @recete/shopify-app build` passed.
- Fixed product setup wizard progression bug in `packages/shopify-app/app/routes/app.products._index.tsx`:
  - Root cause: Step 1 completion (`hasGuidance`) was derived from unsaved draft text.
  - Resulting issue: typing a single character in `Customer instructions` immediately marked step complete and auto-jumped to Step 2.
  - Fix: Step 1 now counts as complete only when guidance is persisted or draft is not dirty (i.e., saved), preventing accidental step jumps before `Continue` / `Save draft`.
- Verification:
  - `pnpm --filter @recete/shopify-app build` passed after fix.
- Updated `packages/shopify-app/app/routes/app.tsx` to avoid duplicate progress indicators on product setup detail pages:
  - hides left-sidebar `Launch progress` card when route is `/app/products` with a selected product query.
  - keeps step progress bar only in the main setup panel for that screen.
- Verification:
  - `pnpm --filter @recete/shopify-app build` passed after UI change.
- Updated `packages/shopify-app/app/routes/app.tsx` to avoid duplicate loading indicators:
  - global shell spinner now renders only when `navigation.state === "loading"` (not on `submitting`).
  - prevents double spinner on product setup while page-level submitting spinner is active.
- Verification:
  - `pnpm --filter @recete/shopify-app build` passed after loader change.
- Updated `packages/shopify-app/app/routes/app.products._index.tsx`:
  - added `Extra source URL` field to Step 3 (`Languages`) so merchants can still provide source URL when Step 2 is auto-completed.
  - root cause observed: Step 2 completion is currently tied to `hasAiKnowledge` (`done: hasAiKnowledge`), so existing knowledge can skip direct Step 2 editing.
- Verification:
  - `pnpm --filter @recete/shopify-app build` passed.
- Refactored product setup flow in `packages/shopify-app/app/routes/app.products._index.tsx` to a true 2-step wizard:
  - Step 1: `Customer instructions`
  - Step 2: `Enhancement` (extra source URL + AI knowledge status + language coverage)
  - removed separate Step 3 block and merged language context into Step 2.
  - wizard editor now keeps Step 2 accessible after Step 1 instead of forcing a hidden/auto-skipped intermediate stage.
- Verification:
  - `pnpm --filter @recete/shopify-app build` passed after refactor.
- Fixed duplicated action controls in Step 2 (`Enhancement`) in `packages/shopify-app/app/routes/app.products._index.tsx`:
  - removed duplicated upper `Continue / Save draft` button group.
  - kept a single action button group at the bottom of the step.
- Verification:
  - `pnpm --filter @recete/shopify-app build` passed.
- Fixed Step 2 language coverage regression trigger in `packages/shopify-app/app/routes/app.products._index.tsx`:
  - Root cause: `Continue` in Step 2 always triggered embeddings pipeline, even when enhancement was already complete and no extra source URL was provided.
  - Symptom: coverage could drop from `3/3` to `0/3` after pressing `Continue` on an already-ready product.
  - Fix: added `enhancementNeedsRun` guard; primary button now:
    - runs embeddings only when needed (`!enhancementDone` or URL provided),
    - otherwise acts as `Done` and exits to product list without rerunning pipeline.
- Verification:
  - `pnpm --filter @recete/shopify-app typecheck` passed.
