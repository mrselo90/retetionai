- Fixed Step 2 language coverage regression trigger in `packages/shopify-app/app/routes/app.products._index.tsx`:
  - Root cause: `Continue` in Step 2 always triggered embeddings pipeline, even when enhancement was already complete and no extra source URL was provided.
  - Symptom: coverage could drop from `3/3` to `0/3` after pressing `Continue` on an already-ready product.
  - Fix: added `enhancementNeedsRun` guard; primary button now:
    - runs embeddings only when needed (`!enhancementDone` or URL provided),
    - otherwise acts as `Done` and exits to product list without rerunning pipeline.
