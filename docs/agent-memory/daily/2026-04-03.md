# 2026-04-03

## Context
- Main focus: Shopify side product enrichment parity with standalone capabilities.
- Environment: local workspace (`retention-agent-ai`), API + Shopify shell codepaths.

## Decisions
- Shopify side should support URL-based enrichment in addition to Shopify-origin product data.
- Product creation should trigger auto enrichment asynchronously to reduce first-run setup friction.
- Existing product knowledge must not be overwritten; new URL content should be merged as additive layers.

## Changes
- Added API schema for URL-based enrichment payload:
  - `packages/api/src/schemas/products.ts`
  - `enrichProductFromUrlSchema` with `source_url`.
- Added API route:
  - `POST /api/products/:id/enrich-from-url`
  - file: `packages/api/src/routes/products.ts`
  - behavior:
    - scrape from provided URL
    - optional Shopify Admin fallback for password-protected storefront pages
    - enrich via existing LLM enrichment flow
    - persist facts snapshot with `sourceType: scrape_enrich_additional_url`
    - merge into existing `raw_text` and `enriched_text` using additive source-stamped blocks
    - regenerate embeddings and refresh caches.
- Added internal helper for non-destructive merge in API route:
  - `buildMergedLayerText(...)` in `packages/api/src/routes/products.ts`.
- Extended Shopify shell platform client:
  - `scrapeMerchantProductAsync(...)`
  - `enrichMerchantProductFromUrl(...)`
  - file: `packages/shopify-app/app/platform.server.ts`.
- Updated Shopify products screen:
  - on `create`, queue auto enrichment with `scrape-async` when product is created
  - new card-level form: `Extra enrichment URL` + `Enrich from URL`
  - file: `packages/shopify-app/app/routes/app.products._index.tsx`.
