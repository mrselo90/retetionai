### Product Lifecycle Analysis
- Deep analysis requested for the full product mapping/setup lifecycle:
  - product selection / mapping
  - creation
  - instruction save
  - enrich / embeddings
  - preview
  - reset knowledge
  - delete
- Confirmed current architecture:
  - single Shopify route `packages/shopify-app/app/routes/app.products._index.tsx`
  - route action fans out through `packages/shopify-app/app/platform.server.ts`
  - backend endpoints in `packages/api/src/routes/products.ts`
- Key findings:
  - `save-setup` persists guidance, then immediately auto-scrapes and optionally enriches from extra URL in the same request cycle
  - `embeddings` can also call `enrich-from-url`, then separately call `generate-embeddings`
  - backend `scrape` route already auto-generates embeddings, so frontend still performs partly duplicated orchestration
  - reset only deletes `knowledge_chunks`; it does not clear `raw_text`, `enriched_text`, facts, or instructions
  - delete removes the local Recete product record; Shopify product remains
  - readiness in UI is mainly driven by `hasGuidance` + `chunkCount/knowledge`, with recent-action polling layered on top
